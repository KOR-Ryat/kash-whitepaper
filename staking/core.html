Core Principles

KASH 스테이킹 메커니즘은 참여자들에게 공정하고 지속 가능한 보상을 제공하기 위해 몇 가지 핵심 원칙을 기반으로 설계되었습니다. 이러한 원칙들은 보상 분배의 기준과 스테이킹 관리 방식을 명확히 정의합니다.

Epoch

KASH 스테이킹 보상은 실시간, 연속적으로 발생하는 것이 아니라 에포크(Epoch) 라고 하는 미리 정의된 시간 주기에 따라 분배됩니다. 각 에포크가 종료되는 시점에 해당 에포크 동안 발생한 총 보상이 계산되어 스테이킹 참여자들에게 할당됩니다.

  • Genesis Timestamp

    • 생태계의 스테이킹이 오픈되는 시점과, 보상 분배가 시작되는 날짜는 다릅니다. 보상 분배가 시작되기 전에 스테이킹을 하여 높은 Early Staker Bonus를 확보할 수 있지만, 실제 이자가 분배되는 Epoch는 이후에 시작됩니다.
    • TOpenT_{Open}TOpen​ : 스테이킹이 오픈되는 시점
    • TStartT_{Start}TStart​ : 보상 분배가 시작되는 시점, (TOpenT_{Open}TOpen​ + STAKING_EPOCH_DELAY)

  • Epoch Reward

    • 각 에포크에 분배될 총 보상액은 에포크마다 성장률(ggg)을 적용하여 점진적으로 증가하도록 설계되었습니다. 이는 초기 에포크보다 후기 에포크에 더 많은 보상이 분배되도록 하여, 장기적인 참여를 독려하고 생태계 성장에 따른 보상 증가를 반영하기 위함입니다.

    • NepochsN_{epochs}Nepochs​ : 전체 스테이킹 기간 동안의 총 에포크 수

      Nepochs=DurationTotalDurationEpochN_{epochs} = \frac{Duration_{Total}}{Duration_{Epoch}}Nepochs​=DurationEpoch​DurationTotal​​
      • DurationTotalDuration_{Total}DurationTotal​: 1차 KASH 생태계의 전체 스테이킹 기간. (상수: #STAKING_TOTAL_DURATION)
      • DurationEpochDuration_{Epoch}DurationEpoch​: 에포크 주기. (상수: #STAKING_EPOCH_DURATION)

    • RewardBaseReward_{Base}RewardBase​ : 첫 에포크 기본 보상액은 전체 보상 총액(RewardTotalReward_{Total}RewardTotal​)과 에포크당 보상 성장률(ggg)을 고려하여, 첫 번째 에포크에 분배될 보상액을 결정합니다.

      RewardBase=RewardTotal×g(1+g)Nepochs−1Reward_{Base} = \frac{Reward_{Total} \times g}{(1 + g)^{N_{epochs}} - 1}RewardBase​=(1+g)Nepochs​−1RewardTotal​×g​
      • RewardTotalReward_{Total}RewardTotal​: 1차 KASH 생태계에서 스테이킹 풀에 분배된 토큰 총량.
      • ggg: 에포크당 보상 성장률. 0 이상. 이 값은 상수 STAKING_REWARD_GROWTH_RATE로 정의됩니다. (Appendix A 참조)

    • RewardEpochReward_{Epoch}RewardEpoch​: kkk번째 에포크에 분배될 총 보상은 다음과 같습니다. (1≤k≤Nepochs1 \le k \le N_{epochs}1≤k≤Nepochs​)

      RewardEpoch=RewardBase×(1+g)k−1Reward_{Epoch} = Reward_{Base} \times (1 + g)^{k-1}RewardEpoch​=RewardBase​×(1+g)k−1

    (현재 설정에 따른 epoch별 보상액 예상 보여주기)

Early Staker Bonus

  • 다음과 같이 스테이킹 시작 시점에 따라 차등적인 보너스 팩터가 각 스테이킹 포지션에 부여됩니다.
EarlyStakerBonus=BaseBonus∗ln(BonusDecayTposition−Topen+1)Early Staker Bonus = Base Bonus * ln( \frac{Bonus Decay}{T_{position} - T_{open} + 1} )EarlyStakerBonus=BaseBonus∗ln(Tposition​−Topen​+1BonusDecay​)
  • TpositionT_{position}Tposition​ : 포지션 생성 (유저의 스테이킹) 시점 (블록 타임스탬프)
  • TopenT_{open}Topen​ : 스테이킹 오픈 시점 (블록 타임스탬프) #STAKING_OPEN_TIMESTAMP
  • Base Bonus : 기준 보너스 값 #STAKING_BONUS_BASE
  • Bonus Decay : 생성 시점에 따른 감소량을 조절하는 값 #STAKING_BONUS_DECAY

이 값은 사용자가 최초로 해당 포지션을 생성(스테이킹)하는 시점에 결정되며, 해당 포지션이 유지되는 동안 변경되지 않습니다.

(현재 설정에 따라, 시간에 따른 Early Staker Bonus 그래프 보여주기)

Staking Reward

  • Weighted Staking Amount

    • 실제 예치량에 보너스 팩터를 적용한 가중 스테이킹 수량이 보상 계산의 기준이 됩니다.
    WeightedStakingAmount=ActualStakingAmount∗(1+EarlyStakerBonus)Weighted Staking Amount = Actual Staking Amount * (1 + Early Staker Bonus)WeightedStakingAmount=ActualStakingAmount∗(1+EarlyStakerBonus)
    • Actual Staking Amount : 포지션에 예치한 토큰 수량

  • Position Staking Reward

    • 개별 포지션의 스테이킹 보상은 에포크 보상을 지분에 따라 분배합니다.
    RewardPosition=Weighted Staking AmountPosition∑Weighted Staking AmountAll Positions∗RewardEpoch,k \text{Reward}_{\text{Position}} = \frac{\text{Weighted Staking Amount}_{\text{Position}}}{\sum \text{Weighted Staking Amount}_{\text{All Positions}}} * \text{Reward}_{\text{Epoch,k}}RewardPosition​=∑Weighted Staking AmountAll Positions​Weighted Staking AmountPosition​​∗RewardEpoch,k​

Staking Position

사용자의 스테이킹 활동은 개별적인 스테이킹 포지션(Staking Position) 단위로 기록 및 관리됩니다. 각 포지션은 독립적인 스테이킹 계약으로 간주될 수 있습니다. 사용자는 여러 개의 포지션을 동시에 보유할 수 있습니다.

  • 포지션 속성

    • Actual Staking Amount : 현재 실제 스테이킹된 토큰 수량
    • Early Staker Bonus : 최초 스테이킹 시 결정된 보너스 팩터
    • Auto Compounding : 자동 복리 기능 활성화 여부
    • Weighted Staking Amount : 계산된 가중 스테이킹 수량

  • NFT (선택사항)

    • 각 포지션은 NFT로 발급되어 소유자에게 전달됩니다.
    • NFT를 직접 거래함으로서 사용자는 Early Staker Bonus를 잃지 않으면서 포지션을 청산할 수 있습니다.
    • NFT 전송시, 전송 시점에 보류중인 이자는 원래 소유자에게 전달됩니다.

  • 포지션 변경

    • 사용자는 언제든지 포지션에 예치된 수량의 일부 또는 전체를 인출할 수 있습니다. 이 때 포지션이 완전히 청산되지 않으면, Early Staker Bonus는 유지됩니다.
    • 기존 포지션에 KASH 토큰을 추가로 예치하는 것은 불가능합니다. 단 Auto Staking을 사용하는 경우에 한해 해당 포지션에서 발생한 이자를 추가하는 것은 가능합니다.
    • 추가로 스테이킹을 수행하는 경우, 해당 시점에 해당하는 별도의 Early Staker Bonus가 적용되는 신규 포지션이 생성됩니다.

  • Switching Fee

    • 사용자는 포지션의 이자를 즉시 수령과 자동 재스테이킹 사이에서 변경할 수 있습니다.
    • 최초 포지션 생성 이후 이를 변경할 때, 현재까지 발생한 이자는 과거 설정에 따라 정산되며 포지션 규모에 비례하여 Switching Fee가 발생합니다.
    Feeswitch=WeightedStakingAmount×FeeRateswitchFee_{switch} = {Weighted Staking Amount} \times FeeRate_{switch}Feeswitch​=WeightedStakingAmount×FeeRateswitch​
    • FeeRateswitchFeeRate_{switch}FeeRateswitch​: Switching Fee의 비율 (상수: #STAKING_SWITCHING_FEE)
